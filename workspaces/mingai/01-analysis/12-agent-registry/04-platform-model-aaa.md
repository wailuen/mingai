## 1. Platform Model Analysis

### 1.1 Role Identification

**Producers** — create value that others consume:

- **Selling Organizations** (sales agents): publish capabilities, respond to inquiries, accept orders, deliver services. They produce the supply side of the marketplace.
- **Buying Organizations** (procurement agents): generate demand in the form of RFQs, POs, and service requests. They produce the demand signals that make the supply side valuable.
- **mingai**: produces the registry infrastructure — agent discovery, trust verification, blockchain ledger, reconciliation tooling. Without mingai's infrastructure, agents have no venue to meet and transact.
- **Third-Party Developers**: produce compatible agents from non-mingai platforms. They expand the network supply without mingai needing to build every agent.

**Consumers** — receive and use the value:

- **Buying Organizations**: consume the discovery and transaction capabilities. They find suppliers and execute procurement without manual effort.
- **Selling Organizations**: consume the inbound demand generated by buying agents. They receive qualified leads and order flow without outbound sales effort.
- **Both sides consume**: the trust layer (KYB scores, transaction history), the audit trail (blockchain records), and the dispute resolution mechanism.

**Partners** — facilitate transactions:

- **KYB Verification Providers** (Stripe Identity, Jumio, D&B): authenticate legal entity identity; HAR delegates identity verification to them.
- **ERP Systems** (SAP, Oracle, NetSuite): receive reconciled transaction records; HAR acts as a data source for their accounting systems.
- **Payment Networks** (bank APIs, SWIFT GPI): execute the financial settlement for Tier 3 transactions; HAR initiates, payment networks settle.
- **Blockchain Infrastructure** (Hyperledger Fabric consortium, Polygon CDK): provide the immutable ledger; HAR operates on top of their protocol.
- **Google A2A Protocol**: provides the agent communication standard; HAR implements A2A as the transport layer.

### 1.2 Core Transactions

**Transaction 1: Capability Discovery**

```
Buying Agent SEARCHES registry by capability + industry + trust score
  → HAR returns ranked list of matching selling agents (by trust, by price, by SLA)
  → Buying Agent REVIEWS agent cards (capabilities, KYB status, transaction history)
  → Buying Agent SELECTS counterparty → INITIATES first contact
  → HAR RECORDS the discovery event (fee: Tier 1)
```

Value created: Procurement speed from days/weeks (manual supplier search) to seconds (AI-powered discovery).

**Transaction 2: Autonomous Negotiation and Commitment**

```
Buying Agent sends RFQ → HAR routes to Selling Agent
  → Selling Agent evaluates, generates counter-proposal
  → Buying Agent evaluates counter-proposal, accepts or counter-offers
  → (Iterations continue within time limit)
  → Agreement reached → HAR RECORDS commitment on-chain (fee: Tier 2)
  → Both parties receive cryptographically signed transaction record
  → Human approval gate triggered if above configured threshold
```

Value created: Procurement cycle compressed from 5-15 business days (human RFQ process) to minutes. Quality maintained because both agents are operating within their company's configured parameters.

**Transaction 3: Financial Settlement and Reconciliation**

```
Seller fulfils commitment → submits delivery confirmation on-chain
  → Buyer's agent verifies delivery (matches PO)
  → Invoice submitted → Buyer's agent verifies invoice matches PO
  → Payment instruction generated → routes to payment network
  → Payment confirmed → HAR RECORDS settlement on-chain (fee: Tier 3, basis points)
  → Reconciliation agent MAPS blockchain record → ERP entries (both sides)
  → Finance teams review exceptions only (flagged mismatches)
```

Value created: Invoice processing cycle from 30-90 days (traditional AP) to same-day or near-real-time; reconciliation from 5-20 manual hours/month to automated.

**Transaction 4: Trust Accumulation**

```
Completed transactions → trust score update for both agents
  → Dispute rate, response time, completion rate updated
  → Trust score feeds next buyer's agent discovery ranking
  → High-trust agents get better placement → more transactions
  → More transactions → higher trust → compounding advantage
```

Value created: Reputation capital that functions like a business credit score — accumulated over years, difficult to replicate quickly, provides competitive advantage in discovery ranking.

### 1.3 Network Effects

**Cross-side (more buyers → more valuable for sellers)**:
Each new buying organization joining HAR creates demand signals across all registered selling agents. A manufacturing company joining HAR with 50+ active suppliers they want to transact with brings 50+ potential relationships into the network. The selling side benefits non-linearly from each buyer addition.

**Same-side (more sellers → more options for buyers)**:
Each new selling agent increases the probability that any buyer will find a match for their specific requirements. At <50 agents, buyers frequently find no match (low liquidity). At 200+ agents, coverage improves substantially. At 1,000+ agents, the network becomes a comprehensive directory.

**Data flywheel (transaction quality improves over time)**:
As more RFQ negotiations occur in a specific industry segment (e.g., manufacturing procurement for steel), the system accumulates:

- Market-standard price ranges for common commodities
- Standard negotiation terms by industry (Incoterms, payment terms, warranty periods)
- Anomaly detection (a price offer 50% below market is flagged as suspicious)

This data makes AI negotiation agents smarter for all participants. Organizations that join later benefit from all historical negotiation data — reducing the cold-start quality problem.

**Trust flywheel (within organizations)**:
Organizations with a long trust history have higher discovery placement, lower friction with counterparties, and faster transaction approval (human approval threshold can be raised for high-trust agents). This creates a compounding advantage: early joiners accumulate trust faster, making late adoption increasingly disadvantageous.

### 1.4 Network Behavior Coverage

#### Accessibility

**How easily can an organization join and transact?**

- Agent Card creation: guided form wizard (< 2 hours for a technical admin)
- KYB verification: Stripe Identity integration (< 1 business day for straightforward cases)
- A2A endpoint setup: SDK + documentation (2-4 hours for a developer; 1-2 days for enterprise integration)
- First transaction: < 24 hours from registration if counterparty is already on network

**Assessment**: MEDIUM. The technical integration (A2A endpoint) requires a developer. Non-technical admins cannot register an agent from scratch — they need engineering support. This limits self-service accessibility compared to the tenant admin console. The SDK reduces this friction but does not eliminate it.

**Accessibility improvement path**: Pre-built agent templates for the most common B2B transaction types (procurement agent, sales agent, logistics agent) that a non-developer can configure and publish.

#### Engagement

**What information drives ongoing participation?**

- Transaction dashboard: volume, completion rate, revenue generated/saved
- Trust score trend: improving or declining and why
- Pending negotiations: active RFQs awaiting response
- Discovery analytics: how many buyers found my agent this week (visibility into inbound)
- Revenue from transactions: how much has the network generated for me this month
- Network activity: total transactions on HAR, growth trends, new participants

**Assessment**: HIGH. Engagement is inherently built in by the transaction value — organizations have direct monetary incentive to monitor and improve their agent performance.

#### Personalization

**Is the network experience tailored to each participant?**

- Search ranking personalized by industry, geography, language, currency match
- Trust score algorithm accounts for agent-specific transaction history
- Recommendations: "Buyers in your industry frequently also work with these agents" (if enough data)
- Threshold alerts: human approval gate tuned per organization's risk tolerance

**Assessment**: MEDIUM. Discovery ranking is personalized to transaction type and trust level. Deeper personalization (e.g., proactive matchmaking: "This buying agent's RFQ matches your selling agent's capabilities") is a Phase 2 feature.

#### Connection

**What data flows connect the participant to the network?**

- Agent messages: real-time (A2A protocol, webhook-based)
- Transaction events: async (settlement, completion, dispute events via webhook)
- ERP reconciliation: batch or real-time (configurable)
- Trust score updates: near-real-time (after each transaction completion)
- Platform notifications: operational (dispute filed, KYB expiry approaching, network alerts)

**Assessment**: HIGH. All connections are bidirectional and event-driven. The ERP connector makes HAR a data source for financial systems, not a siloed application.

#### Collaboration

**Can participants work together through the platform?**

- AI-to-AI negotiation: direct collaboration between counterparty agents
- Dispute mediation: both parties submit evidence, AI mediator facilitates
- Contract renewal: agent-initiated outreach to existing counterparties
- Reference and reputation: (not yet designed) organizations could endorse each other's agents

**Assessment**: MEDIUM. Direct A2A collaboration is strong (that's the core of the platform). Human-layer collaboration (multiple stakeholders from the same company reviewing a negotiation) is not yet designed.

## 2. AAA Framework

### 2.1 Automate — Reduce Operational Costs

| Manual Process Today                                        | Automated Replacement                                                                    | Cost Saved                                              |
| ----------------------------------------------------------- | ---------------------------------------------------------------------------------------- | ------------------------------------------------------- |
| Manual supplier search and qualification                    | AI agent searches registry → finds KYB-verified agents with matching capabilities        | 4-20 hours/search → seconds                             |
| Human RFQ creation and distribution                         | AI agent generates RFQ from procurement system → routes to matching agents automatically | 2-8 hours → seconds                                     |
| Human price negotiation                                     | AI agent negotiates within configured parameters → escalates exceptions only             | Full-time procurement staff → AI handles routine volume |
| Invoice matching (PO vs invoice)                            | Reconciliation agent matches on-chain transaction → ERP invoice automatically            | 10-30 min/invoice → automated                           |
| Dispute documentation                                       | All transaction state transitions already on-chain → no reconstruction needed            | 5-20 hours of evidence gathering → immediate            |
| New supplier onboarding (document collection, verification) | KYB done at registration → trust score immediate → no per-relationship onboarding        | 2-6 weeks per new supplier → 1-2 days                   |

**Automation score**: 9/10 — the core value proposition IS automation. HAR cannot deliver value without automating the manual B2B transaction process. This is the highest automation potential of any product in the mingai portfolio.

### 2.2 Augment — Reduce Decision-Making Costs

| Decision                                             | Without HAR                                  | With HAR                                                                           | Quality Improvement                            |
| ---------------------------------------------------- | -------------------------------------------- | ---------------------------------------------------------------------------------- | ---------------------------------------------- |
| "Should we work with this supplier?"                 | Manual reference checks, subjective judgment | Trust score + transaction history + dispute rate + attestation depth               | Objective, data-driven, auditable              |
| "Is this price offer fair?"                          | Buyer researches market price manually       | AI agent compares offer against network-wide price distribution for this commodity | Immediate market intelligence                  |
| "Did this invoice match our PO?"                     | Accounts payable team reviews each line      | Reconciliation agent flags exceptions only                                         | 95%+ auto-approved; humans review exceptions   |
| "Should we escalate this dispute?"                   | Legal team reviews case by case              | AI mediator provides recommendation with probability of success                    | Consistent, data-informed escalation decisions |
| "Which suppliers have capacity this month?"          | Phone calls and email                        | Availability check via A2A → availability result in <2s                            | Real-time vs. stale                            |
| "What are standard payment terms for this industry?" | Procurement team relies on experience        | Network-wide analytics on accepted payment terms by industry and deal size         | Data-driven negotiation anchoring              |

**Augmentation score**: 8/10 — HAR provides meaningful decision augmentation for procurement and sales operations. The limitation is that early-stage network data is sparse; augmentation quality improves with transaction volume.

### 2.3 Amplify — Reduce Expertise Costs for Scaling

| Scaling Challenge                                        | Without HAR                                                          | With HAR                                                                                               |
| -------------------------------------------------------- | -------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| Company wants to expand B2B transactions by 10×          | 10× procurement staff required                                       | AI agents handle 10× volume; same human oversight team                                                 |
| Enter new geographic market (e.g., expand from SG to JP) | New supplier relationships, cultural negotiation norms, legal review | Japanese-speaking agent + JP-registered suppliers already on registry; A2A protocol handles language   |
| Process invoices during year-end (10× volume spike)      | Hire temporary AP staff                                              | Reconciliation agent scales automatically; no staffing spike                                           |
| Small suppliers (too small for sales team attention)     | Revenue from small accounts not captured                             | AI agent handles SMB transactions autonomously; no threshold too small                                 |
| Procurement team expert leaves the company               | Institutional knowledge lost                                         | Negotiation history on-chain; AI agent has access to all prior terms negotiated with each counterparty |
| New product launch requiring new supplier categories     | Start supplier search from scratch                                   | Registry search returns suppliers by capability → accelerated discovery                                |

**Amplification score**: 9/10 — HAR's amplification is strongest of any product in the mingai portfolio. The fundamental value of HAR is that it enables organizations to scale B2B commerce without scaling headcount proportionally. This is the highest leverage point: procurement team of 3 handles $10M/year of procurement → with HAR, same team handles $50M+.

| AAA — Automate                        | 9/10       | Core value IS automation; B2B transaction volume scales without headcount                                      |

**Critical caveat**: The 8.3/10 is the steady-state score at network liquidity. At Phase 0 (registry launched, <50 agents):

| Dimension       | Phase 0 Score | Reason                                  |
| --------------- | ------------- | --------------------------------------- |
| Network effects | 2/10          | No liquidity = no value                 |
| AAA — Augment   | 3/10          | No historical data for decision support |
| Overall         | **3.5/10**    | Pre-liquidity phase                     |

The journey from 3.5/10 to 8.3/10 is the central risk and the central opportunity.

### Critical Observation: Network Liquidity is Binary

Unlike the tenant admin console (where value accumulates gradually with each feature), HAR has a **liquidity threshold** — a minimum number of active agents below which the registry has near-zero utility.

Estimated liquidity threshold: ~100 active agents with real transaction history across at least 5 industries.

Below threshold: HAR is a directory (limited value). Above threshold: HAR is a marketplace (high value). The transition is non-linear.

**Investment implication**: HAR requires a concentrated investment to push past the liquidity threshold. Gradual resource allocation will not work — it will keep HAR permanently below threshold.

### Gaps to Address

**Gap 1: Developer friction for agent integration**
The A2A endpoint requirement blocks non-technical organizations from joining. Pre-built agent templates for common scenarios (SAP-connected procurement agent, NetSuite-connected sales agent) would lower the barrier from developer-required to admin-configurable.

**Gap 2: Financial settlement infrastructure not built**
Tier 3 transactions (financial) require payment network integration. This requires banking relationships, regulatory compliance (MSB licensing in some jurisdictions), and significant technical infrastructure. Without Tier 3, HAR's monetization is capped at Tier 1-2 fees (low unit value). Plan must explicitly gate Tier 3 on financial infrastructure buildout.

**Gap 3: Legal standing of AI agent commitments is jurisdiction-specific**
A PO committed by an AI agent may not be legally binding in all jurisdictions without explicit authorization. Different countries have different rules on electronic agent authority. HAR needs per-jurisdiction legal opinion before enabling Tier 2+ transactions in each market.

**Gap 4: Dispute resolution panel does not exist**
The Phase 2+ dispute resolution design relies on a panel of neutral arbitrators. This panel does not exist and cannot be assembled without network participants. Bootstrapping the panel requires either (1) mingai employees as interim arbitrators (conflict of interest risk) or (2) partnership with an ADR (Alternative Dispute Resolution) provider from day 1.
